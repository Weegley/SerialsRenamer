# SerialsRenamer

English version: [README.md](README.md)

Производственный Python-скрипт для нормализации папок сериалов и файлов эпизодов для Jellyfin и похожих медиабиблиотек.

Он помогает привести хаотичную коллекцию сериалов к предсказуемой структуре:

```text
Название сериала (Оригинальное название) (Год) [tmdbid-123456][imdbid-tt1234567]/
└── Season 01/
    ├── S01E01.mkv
    ├── S01E01.forced.rus.srt
    ├── S01E02.mkv
    └── ...
```

## Ключи API и доступы

Скрипт использует:

- [Kinopoisk Api Unofficial](https://kinopoiskapiunofficial.tech)
- [TMDb](https://www.themoviedb.org)

### Kinopoisk Api Unofficial

Сейчас в исходнике для быстрого старта и тестов зашит общий ключ `Kinopoisk Api Unofficial`.

Для пробного запуска это удобно, но для постоянного использования **настоятельно рекомендуется** заменить его на **свой собственный ключ Kinopoisk Api Unofficial**.

Почему:

- общий ключ может упираться в rate limit
- общий ключ может перестать работать в любой момент
- со своим ключом работа предсказуемее и стабильнее

### TMDb

Для TMDb нужно указать свой bearer token в:

```python
DEFAULT_TMDB_BEARER = "..."
```

Если `DEFAULT_TMDB_BEARER` пустой или всё ещё содержит значение-заглушку, скрипт сразу завершится с понятной ошибкой.

Если токен указан, но неверный, скрипт покажет ошибку TMDb API, которую вернул сервер.

На практике TMDb обязателен для нормальной работы, потому что:

- профиль `intl` напрямую работает через TMDb
- профиль `ru` использует TMDb для enrichment, external IDs и локализованных названий
- мультиязычные поля `title_XX` зависят от TMDb translations

Итого:

- встроенный ключ Kinopoisk допустим для быстрого теста
- свой Kinopoisk ключ очень желателен
- свой TMDb bearer token обязателен

---

Скрипт задуман как интерактивный, консервативный и безопасный для файловой системы:

- он умеет показывать план изменений до применения
- он спрашивает подтверждение для каждого сериала
- он не делает агрессивных догадок в неоднозначных случаях
- он не удаляет непустые папки вслепую
- он использует fallback-логику, когда имена файлов грязные, но хотя бы сортируемые

## Возможности

### Профили метаданных

Скрипт поддерживает два профиля метаданных:

- `ru`
- `intl`

#### Профиль `ru`

- основной источник: Kinopoisk
- enrichment: TMDb
- IMDb id: из Kinopoisk, если есть, иначе из TMDb
- основное название обычно русское

Схема резолва:

```text
kp -> tmdb ; imdb = kp if present else tmdb
```

#### Профиль `intl`

- основной источник: TMDb
- IMDb id: из TMDb
- Kinopoisk не используется как основной источник метаданных
- основное название обычно английское / международное

Схема резолва:

```text
tmdb -> imdb
```

### Канонические форматы ID

Скрипт всегда рендерит ID в каноническом Jellyfin-friendly виде:

- `[kp-1234567]`
- `[tmdbid-123456]`
- `[imdbid-tt1234567]`

Старые входные формы тоже распознаются при парсинге.

Поддерживаемые входные варианты:

- Kinopoisk:
  - `[kp1234567]`
  - `[kp-1234567]`

- TMDb:
  - `[tmdb12345]`
  - `[tmdb-12345]`
  - `[tmdbid-12345]`

- IMDb:
  - `[tt1234567]`
  - `[imdbidtt1234567]`
  - `[imdbid=tt1234567]`
  - `[imdbid-tt1234567]`

### Распознавание сериалов

- ищет сериал через Kinopoisk или TMDb в зависимости от профиля
- поддерживает прямой ввод `kp` id при ручном выборе через Kinopoisk
- поддерживает прямой ввод `tmdb` id при ручном выборе через TMDb
- повторно использует ID, уже встроенные в имена папок
- поддерживает tolerant parsing старых и новых форматов ID

### Нормализация папок

- нормализует имя корневой папки сериала по шаблонам
- нормализует имена сезонных папок
- нормализует имена файлов эпизодов и субтитров
- умеет переименовывать уже распознанные, но плохо названные исходные папки
- умеет работать в безопасном metadata-only режиме для уже нормализованных корней сериалов

### Режим обновления

`--update-series-folders`

Этот режим:

- проходит по уже нормализованным корневым папкам сериалов
- добирает недостающие метаданные и ID
- при необходимости переименовывает только корневую папку сериала
- **не** перемещает файлы эпизодов и субтитров

Это безопасный режим обновления метаданных.

### Распознавание эпизодов

Распознаёт типовые паттерны эпизодов:

- `S01E01`
- `S1E1`
- `S01.E01`
- `S01-E01`
- `S01_E01`
- `1x01`
- `01x01`
- `1x1`
- `01x1`
- `E01`, если сезон уже известен
- номер в начале имени, например:
  - `01 Episode title...`
  - `01.Series.Name...`

### Распознавание сезонов

Распознаёт типовые форматы сезонных папок:

- `Season 1`
- `Season 01`
- `1 season`
- `S1`
- `S01`
- имена папок, оканчивающиеся номером сезона, например:
  - `sopranos1`
  - `sopranos6`

Также есть fallback-допущение:

- если файлы лежат прямо в корне сериала и сезон неизвестен,
  они считаются относящимися к `Season 01`

### Поддержка субтитров

Поддерживаемые форматы субтитров:

- `.srt`
- `.ass`
- `.ssa`
- `.sub`

Старое распознавание метаданных субтитров по-прежнему понимает типовые токены:

- язык:
  - `ru`, `rus`, `russian`
  - `en`, `eng`, `english`
- тип:
  - `forced`
  - `full`
  - `sdh`

### Сохранение subtitle suffix

Внешние субтитры теперь обрабатываются консервативно:

- за базу берётся имя соответствующего видеофайла
- всё, что идёт после stem видео, считается subtitle suffix
- suffix сохраняется в итоговом имени субтитра
- разделители в suffix нормализуются к точкам

Примеры:

- `Show S01E01.srt` → `S01E01.srt`
- `Show S01E01_ru.srt` → `S01E01.ru.srt`
- `Show S01E01.en.srt` → `S01E01.en.srt`
- `Show S01E01_forced.rus.srt` → `S01E01.forced.rus.srt`
- `Show S01E01_full.V1.rus.srt` → `S01E01.full.V1.rus.srt`

Это позволяет сохранить полезную suffix-информацию без попытки полностью “понять” каждую экзотическую схему именования субтитров.

### Безопасная fallback-логика

Когда в именах файлов нет явно распознаваемого номера эпизода:

- видеофайлы назначаются по отсортированному порядку внутри исходной папки
- нумерация начинается с `E01`
- уже занятые целевые имена пропускаются
- парные субтитры ищутся относительно соответствующего stem видео
- suffix таких субтитров сохраняется и переносится вместе с видео

Это делает скрипт устойчивым к плохо названным релизам без попытки слишком агрессивно угадывать экзотические схемы именования.

### Консервативное поведение

- неоднозначные subtitle-only файлы не угадываются слишком агрессивно
- непустые папки не удаляются
- уже нормализованные сериалы не трогаются
- `Ctrl+C` или пункт меню `0` завершают работу корректно
- ошибки записи cache и log на проблемных путях обрабатываются мягче

## Использование

### Базовый запуск

```bash
python SerialsRenamer.py /path/to/Serials
```

### Только предпросмотр изменений

```bash
python SerialsRenamer.py /path/to/Serials --dry-run
```

### Интерактивный предпросмотр перед каждым сериалом

```bash
python SerialsRenamer.py /path/to/Serials --dry-first
```

### Использовать международный профиль метаданных

```bash
python SerialsRenamer.py /path/to/Serials --metadata-profile intl
```

### Всегда подтверждать результаты поиска вручную

```bash
python SerialsRenamer.py /path/to/Serials --mode manual
```

### Обновить только уже нормализованные корневые папки сериалов

```bash
python SerialsRenamer.py /path/to/Serials --update-series-folders --dry-run
```

## Интерактивный сценарий работы

Для каждого найденного сериала скрипт:

1. сканирует файлы
2. пытается вывести название из структуры папок
3. резолвит метаданные сериала по активному профилю
4. строит план переименования и перемещения
5. при необходимости показывает preview
6. применяет изменения после подтверждения

### Меню выбора результата поиска

Типовые варианты:

- `0` — выйти из всего скрипта
- `98` — пропустить текущий экран выбора
- `99` — повторить тот же поиск
- `Enter` — принять единственный результат, если он один
- любой другой текст — новый поисковый запрос или прямой `kp` / `tmdb` id

Примеры валидного ручного ввода:

- `kp1234567`
- `1234567`
- `tmdb1215`
- `The Sopranos`

```text

```

## Логирование

Скрипт пишет человекочитаемый log операций.

Файл по умолчанию:

```text
SerialsRenamer.operations.log
```

Типовые записи:

```text
============================================================
SERIES  Sopranos(full version)  ->  Блудливая Калифорния (Californication) (2007) [kp-394375][tmdbid-1215][imdbid-tt0904208]

MOVE   sopranos1/The Sopranos [S01E01, Goblin].avi  =>  Season 01/S01E01.avi
PAIRSB plevako/01 Плевако .2023.WEB-DLRip.Files-x_forced.rus.srt  =>  Season 01/S01E01.forced.rus.srt
FALLBK sopranos6/Klan_Soprano_VI_13_DVDRip.avi  =>  Season 06/S06E13.avi
DELDIR old_subfolder
KEEPDIR leftovers  ::  not empty
SUMMARY ops=18 removed_dirs=5 kept_dirs=1 errors=0
```

### Типы операций

- `RENAME` — переименование корневой папки сериала
- `MOVE` — обычное распознанное перемещение
- `FALLBK` — fallback-перемещение по отсортированному списку видео
- `PAIRSB` — субтитр, перемещённый вместе с fallback-парой
- `DELDIR` — удалена пустая папка
- `KEEPDIR` — папка оставлена, потому что она не пуста
- `ERROR` — операция завершилась ошибкой

## Cache

Скрипт кэширует разрешённые метаданные, чтобы уменьшить количество повторной ручной работы.

Файл cache по умолчанию:

```text
.series_rename_cache.json
```

В cache могут храниться:

- выбранное название
- Kinopoisk ID
- TMDb ID
- IMDb ID
- год
- original title
- localized titles

При тестировании новых шаблонов или логики локализаций рекомендуется использовать новый cache-файл или удалить старый.

Если сохранение cache не удалось из-за сетевого пути или прав доступа, скрипт покажет предупреждение и завершится без traceback.

## Шаблоны именования

Скрипт использует шаблоны для итоговой структуры.

### Шаблоны по умолчанию

```python
RU_SERIES_FOLDER_TEMPLATE = "{title} ({original_title}) ({year}) [kp-{kp}][tmdbid-{tmdb}][imdbid-{tt}]"
INTL_SERIES_FOLDER_TEMPLATE = "{title} ({original_title}) ({year}) [tmdbid-{tmdb}][imdbid-{tt}]"
RU_SEASON_FOLDER_TEMPLATE = "Season {season:02d}"
INTL_SEASON_FOLDER_TEMPLATE = "Season {season:02d}"
RU_EPISODE_FILE_TEMPLATE = "{title} ({original_title}) S{season:02d}E{episode:02d}{ext}"
INTL_EPISODE_FILE_TEMPLATE = "S{season:02d}E{episode:02d}{ext}"
RU_SUBTITLE_FILE_TEMPLATE = "{title} ({original_title}) S{season:02d}E{episode:02d}{sub_suffix}{ext}"
INTL_SUBTITLE_FILE_TEMPLATE = "S{season:02d}E{episode:02d}{sub_suffix}{ext}"
```

### Доступные поля шаблонов

Основные поля:

- `{title}` — основное отображаемое название для активного профиля метаданных
- `{original_title}` — оригинальное или provider-original название
- `{title_local}` — локализованное название для активного профиля

Локализованные поля названий:

- `{title_ru}`
- `{title_en}`
- `{title_de}`
- `{title_fr}`
- `{title_ko}`
- другие поля вида `title_XX`, включая региональные варианты вроде `{title_pt_BR}` или `{title_zh_CN}`

Остальные поля:

- `{kp}` — Kinopoisk ID без префикса
- `{tmdb}` — TMDb ID без префикса
- `{tt}` — IMDb ID вместе с `tt`
- `{year}` — год выхода
- `{season}` — номер сезона
- `{episode}` — номер эпизода
- `{lang}` — language token субтитра
- `{subtype}` — subtype token субтитра
- `{sub_suffix}` — сохранённый suffix субтитра после matching video stem
- `{ext}` — расширение файла вместе с точкой

### Примечания по `{sub_suffix}`

`{sub_suffix}` предназначен для subtitle template.

Он сохраняет “сырой” suffix, который идёт после имени соответствующего видеофайла, и нормализует разделители к точкам.

Примеры:

- пустой suffix → `""`
- `_ru` → `.ru`
- `.en` → `.en`
- `_forced.rus` → `.forced.rus`
- `_full.V1.rus` → `.full.V1.rus`

### Правила дедупликации

`{title}` всегда считается главным полем.

Перед рендером:

- если `{original_title}` совпадает с `{title}`, оно очищается
- если любой `{title_XX}` совпадает с `{title}`, он очищается
- если вторичные поля дублируют друг друга, сохраняется только первое уникальное значение, остальные очищаются

Дедупликация использует нормализованные title keys:

- без учёта регистра
- с игнорированием повторных пробелов
- с нормализацией разделителей и пунктуации

### Примеры пользовательских шаблонов

Консервативный `intl`-вариант с дополнительным русским названием:

```python
INTL_SERIES_FOLDER_TEMPLATE = "{title} ({title_ru}) ({year}) [tmdbid-{tmdb}][imdbid-{tt}]"
```

Более подробный мультиязычный пример:

```python
INTL_SERIES_FOLDER_TEMPLATE = "{title} ({original_title}) ({title_ru}) ({title_fr}) ({year}) [tmdbid-{tmdb}][imdbid-{tt}]"
```

Скрипт валидирует поля шаблонов при старте и завершится с ошибкой, если в шаблоне есть неизвестное поле.

## Правила безопасности

Скрипт намеренно консервативен.

### Что он делает

- переименовывает и реорганизует распознанные файлы
- использует fallback только для сортируемых неизвестных видеофайлов
- сохраняет subtitle suffix, если найден matching video stem
- перемещает парные субтитры вместе с fallback-видео
- удаляет пустые исходные папки
- оставляет непустые папки на месте
- в режиме `--update-series-folders` обновляет только корневые папки сериалов

### Чего он не делает

- не угадывает вслепую произвольные subtitle-only файлы
- не удаляет непустые папки
- не перезаписывает существующие целевые файлы
- не пытается агрессивно понимать каждую странную схему именования, встреченную в природе

## Рекомендуемый сценарий работы

Для новых коллекций:

```bash
python SerialsRenamer.py /path/to/Serials --dry-first
```

Это самый безопасный режим:

- вы видите план до применения
- можно пропустить сериал или повторить поиск
- можно остановить работу в любой момент через `0` или `Ctrl+C`

Когда вы уже доверяете текущей партии:

```bash
python SerialsRenamer.py /path/to/Serials
```

## Ограничения

- зависит от доступности Kinopoisk и TMDb
- не пытается понимать все возможные кастомные фанатские схемы именования
- неоднозначные subtitle-only коллекции намеренно не угадываются слишком агрессивно
- римские цифры в номерах эпизодов напрямую не поддерживаются, если только sortable fallback не оказывается достаточным

---

## Поведение при выходе

- `0` в меню корректно завершает весь скрипт
- `Ctrl+C` тоже завершает работу корректно
- cache сохраняется при выходе, если это возможно
- ошибки сохранения cache понижаются до warning
- финализация operations log на проблемных путях стала безопаснее
